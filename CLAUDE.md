# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common Commands

```bash
# Build provider binary + regenerate schema.json + install to Pulumi plugin cache
make provider

# Build provider + generate Go SDK
make build

# Lint (requires golangci-lint)
make lint
make lint-fix

# Tidy all go.mod files (root, sdk/, tests/, examples/go/)
make ensure

# Update all dependencies to latest and tidy
make ensure-update

# Run provider tests
make test_provider

# Regenerate Go SDK from the built binary
make sdk_go

# Preview / apply / destroy the YAML example stack
make plan
make up
make down
```

`make provider` is the most common inner-loop command — it builds, regenerates `schema.json`, and installs the plugin locally in one step. `VERSION` is derived from `pulumictl get version`; it must be set or `pulumictl` must be in PATH for the build to succeed.

## Module Layout

This is a **multi-module repository**. Each subdirectory has its own `go.mod`:

| Directory | Purpose |
|-----------|---------|
| `.` (root) | Provider implementation — the only module you normally edit |
| `sdk/` | Generated Go SDK — do not hand-edit; regenerate with `make sdk_go` |
| `tests/` | Provider acceptance tests |
| `examples/go/` | Go example program |

The root `go.mod` has a `replace` block that redirects `github.com/dexidp/dex` to NetBird's fork (`github.com/netbirdio/dex`). This is required because Go replace directives in dependencies do not propagate — without it `go mod tidy` fails with "module does not contain package `github.com/dexidp/dex/server/signer`".

## Architecture

### Provider registration (`provider/provider.go`)

All 15 resources are registered in a single `infer.Provider(...)` call. Adding a new resource requires: (1) implementing it under `provider/resource/`, (2) adding it to the `Resources` slice in `provider/provider.go`.

### Resource implementation pattern

Every resource lives in `provider/resource/<name>.go` and implements these methods on a zero-value struct:

- **`Create`** — dry-run aware; returns a stable ID (usually the API-assigned ID)
- **`Read`** — reconciles live API state into both `Inputs` and `State`
- **`Update`** — calls the API update endpoint; never mutates immutable fields
- **`Delete`** — calls the API delete endpoint
- **`Diff`** — builds a `map[string]p.PropertyDiff` by comparing `req.Inputs` vs `req.State`; use `p.UpdateReplace` for immutable fields
- **`Check`** — validates inputs before any API call; append to `failures` rather than returning an error
- **`WireDependencies`** — explicitly maps every input field to its output field

Each resource has a mirrored pair of structs: `<Name>Args` (inputs) and `<Name>State` (outputs). Both implement `Annotate(infer.Annotator)` to add schema descriptions and defaults.

### NetBird API client (`provider/config/`)

`config.GetNetBirdClient(ctx)` returns a `rest.Client` configured from the provider config (`NetBirdURL` + `NetBirdToken`). All resource methods call this at the start. The client is the generated NetBird REST client from `github.com/netbirdio/netbird/shared/management/client/rest`.

### Type conversions

The `provider/resource/` package uses `nbapi` (aliased import of `github.com/netbirdio/netbird/shared/management/http/api`) for all API types. Convert between Pulumi structs and API types with local helpers — see `common.go` for the `to/fromAPIResource*` pattern and `util.go` for pointer/slice helpers (`equalPtr[T]`, `equalSlice`, `equalSlicePtr`, `strPtr`, `boolVal`, `isBlank`, `isNotFoundErr`, `parseNestedID`, `sortedStrings`).

### Enums

Declare a named `string` type, define `const` values backed by `nbapi` constants, and implement `Values() []infer.EnumValue[T]` to register the enum in the schema. See `Type` in `common.go` or `RuleAction`/`Protocol` in `policy.go` for examples.

### Schema generation

`schema.json` is auto-generated by running `pulumi package get-schema` against the built binary (done automatically by `make provider`). Never hand-edit `schema.json`; it is committed only to track drift in CI.

## Established patterns

### Remote deletion / drift (Read and Delete)

Every `Read` must handle resources deleted outside Pulumi. Use `isNotFoundErr(err)` from `util.go` and return an empty ID to signal Pulumi to remove the resource from state:

```go
resource, err := client.Something.Get(ctx, req.ID)
if err != nil {
    if isNotFoundErr(err) {
        return infer.ReadResponse[FooArgs, FooState]{
            ID:     "",
            Inputs: FooArgs{},  //nolint:exhaustruct
            State:  FooState{}, //nolint:exhaustruct
        }, nil
    }
    return infer.ReadResponse[FooArgs, FooState]{}, fmt.Errorf("reading foo failed: %w", err)
}
```

Every `Delete` must treat 404 as success (idempotent destroy):

```go
err = client.Something.Delete(ctx, req.ID)
if err != nil && !isNotFoundErr(err) {
    return infer.DeleteResponse{}, fmt.Errorf("deleting foo failed: %w", err)
}
```

The `nolint:exhaustruct` comment is required on zero-value `Args`/`State` literals inside `infer.ReadResponse` because `exhaustruct` applies to all types in this package, not just `nbapi` types.

### Optional field diffs

Always compare optional pointer fields unconditionally — never guard with `!= nil`:

```go
// CORRECT — detects removal (nil input clears a previously set value)
if !equalPtr(req.Inputs.Description, req.State.Description) { ... }

// WRONG — misses the case where user removes the field
if req.Inputs.Description != nil && !equalPtr(...) { ... }
```

### Slice handling — never sort req.Inputs or req.State in place

Sorting input/state slices mutates data shared across lifecycle hooks and makes diagnostics unreliable. Always clone first:

```go
// CORRECT
sorted := sortedStrings(req.Inputs.GroupIDs)  // sortedStrings clones then sorts

// WRONG
slices.Sort(req.Inputs.GroupIDs)
```

Use `equalSlice` (order-insensitive comparison) in `Diff` rather than sorting both sides in place.

### Compound import IDs for nested resources

Resources with a parent (NetworkResource, NetworkRouter) support `parentID/childID` import IDs. Use `parseNestedID` from `util.go` — it validates both parts are non-empty and returns a clear error if the format is wrong:

```go
if networkID == "" {
    var parseErr error

    networkID, resourceID, parseErr = parseNestedID("NetworkResource", req.ID)
    if parseErr != nil {
        return infer.ReadResponse[...]{}, parseErr
    }
}
```

### Diff kind for parent/network IDs

Fields that identify a parent resource (e.g. `networkID` on NetworkResource, NetworkRouter, Route) must use `p.UpdateReplace`, not `p.Update`, because changing the parent requires destroying and re-creating the child.

### Shared helpers in common.go

`common.go` holds helpers shared across multiple resources:

- `toAPIResource` / `fromAPIResource` / `toAPIResourceList` / `fromAPIResourceList` — convert between `Resource` and `nbapi.Resource`
- `equalResourcesPtr` / `sortedResources` / `compareResources` — compare `[]Resource` slices by value, sorted by type+ID

Do not duplicate these in individual resource files.

### Check validation completeness

Every `Check` method must validate:

- Required string fields are non-blank (`isBlank`)
- Optional slice fields have no blank entries (iterate with `fmt.Sprintf("field[%d]", i)` property paths)
- Mutually exclusive fields (e.g. `network` vs `domains`, `peer` vs `peerGroups`)
- At-least-one constraints (e.g. sources or sourceResource required)

Extract a helper function (e.g. `fooCheckArgs`) when the validation body exceeds ~30 cyclomatic complexity to satisfy `gocognit`/`cyclop` linters; annotate with `//nolint:cyclop,gocognit` on that helper.

## Linting

The project uses `golangci-lint` with `default: all` and selectively disables linters. Notable active linters:

- **`exhaustruct`** — all struct literals for types in `nbapi` must fill every field; use `nil`/zero for optional fields rather than omitting them. Our own `Args`/`State` types are also subject to this rule — use `//nolint:exhaustruct` only for intentional zero-value returns (e.g. the 404 read-miss pattern above).
- **`wrapcheck`** — errors from external packages must be wrapped with `fmt.Errorf("...: %w", err)`.
- **`nlreturn`** — blank line required before `return` statements; also required before `if` blocks that follow variable declarations (`wsl_v5`).
- **`nonamedreturns`** — do not use named return values.
- **`gomoddirectives`** disabled — the dex `replace` directives in `go.mod` are intentional.

Run `GOFLAGS=-buildvcs=false golangci-lint run -c ./.golangci.yml` (or `make lint`) — the `GOFLAGS` flag is required because the golangci-lint container doesn't have git history.
