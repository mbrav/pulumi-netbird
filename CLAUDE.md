# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common Commands

```bash
# Build provider binary + regenerate schema.json + install to Pulumi plugin cache
make provider

# Build provider + generate Go SDK
make build

# Lint (requires golangci-lint)
make lint
make lint-fix

# Tidy all go.mod files (root, sdk/, tests/, examples/go/)
make ensure

# Update all dependencies to latest and tidy
make ensure-update

# Run provider tests
make test_provider

# Regenerate Go SDK from the built binary
make sdk_go

# Preview / apply / destroy the YAML example stack
make plan
make up
make down
```

`make provider` is the most common inner-loop command ŌĆö it builds, regenerates `schema.json`, and installs the plugin locally in one step. `VERSION` is derived from `pulumictl get version`; it must be set or `pulumictl` must be in PATH for the build to succeed.

## Module Layout

This is a **multi-module repository**. Each subdirectory has its own `go.mod`:

| Directory | Purpose |
|-----------|---------|
| `.` (root) | Provider implementation ŌĆö the only module you normally edit |
| `sdk/` | Generated Go SDK ŌĆö do not hand-edit; regenerate with `make sdk_go` |
| `tests/` | Provider acceptance tests |
| `examples/go/` | Go example program |

The root `go.mod` has a `replace` block that redirects `github.com/dexidp/dex` to NetBird's fork (`github.com/netbirdio/dex`). This is required because Go replace directives in dependencies do not propagate ŌĆö without it `go mod tidy` fails with "module does not contain package `github.com/dexidp/dex/server/signer`".

## Architecture

### Provider registration (`provider/provider.go`)

All 15 resources are registered in a single `infer.Provider(...)` call. Adding a new resource requires: (1) implementing it under `provider/resource/`, (2) adding it to the `Resources` slice in `provider/provider.go`.

### Resource implementation pattern

Every resource lives in `provider/resource/<name>.go` and implements these methods on a zero-value struct:

- **`Create`** ŌĆö dry-run aware; returns a stable ID (usually the API-assigned ID)
- **`Read`** ŌĆö reconciles live API state into both `Inputs` and `State`
- **`Update`** ŌĆö calls the API update endpoint; never mutates immutable fields
- **`Delete`** ŌĆö calls the API delete endpoint
- **`Diff`** ŌĆö builds a `map[string]p.PropertyDiff` by comparing `req.Inputs` vs `req.State`; use `p.UpdateReplace` for immutable fields
- **`Check`** ŌĆö validates inputs before any API call; append to `failures` rather than returning an error
- **`WireDependencies`** ŌĆö explicitly maps every input field to its output field

Each resource has a mirrored pair of structs: `<Name>Args` (inputs) and `<Name>State` (outputs). Both implement `Annotate(infer.Annotator)` to add schema descriptions and defaults.

### NetBird API client (`provider/config/`)

`config.GetNetBirdClient(ctx)` returns a `rest.Client` configured from the provider config (`NetBirdURL` + `NetBirdToken`). All resource methods call this at the start. The client is the generated NetBird REST client from `github.com/netbirdio/netbird/shared/management/client/rest`.

### Type conversions

The `provider/resource/` package uses `nbapi` (aliased import of `github.com/netbirdio/netbird/shared/management/http/api`) for all API types. Convert between Pulumi structs and API types with local helpers ŌĆö see `common.go` for the `to/fromAPIResource*` pattern and `util.go` for pointer/slice helpers (`equalPtr[T]`, `equalSlice`, `strPtr`, `boolVal`, `isBlank`).

### Enums

Declare a named `string` type, define `const` values backed by `nbapi` constants, and implement `Values() []infer.EnumValue[T]` to register the enum in the schema. See `Type` in `common.go` or `RuleAction`/`Protocol` in `policy.go` for examples.

### Schema generation

`schema.json` is auto-generated by running `pulumi package get-schema` against the built binary (done automatically by `make provider`). Never hand-edit `schema.json`; it is committed only to track drift in CI.

## Linting

The project uses `golangci-lint` with `default: all` and selectively disables linters. Notable active linters:

- **`exhaustruct`** ŌĆö all struct literals for types in `nbapi` must fill every field; use `nil`/zero for optional fields rather than omitting them.
- **`wrapcheck`** ŌĆö errors from external packages must be wrapped with `fmt.Errorf("...: %w", err)`.
- **`gomoddirectives`** disabled ŌĆö the dex `replace` directives in `go.mod` are intentional.

Run `GOFLAGS=-buildvcs=false golangci-lint run -c ./.golangci.yml` (or `make lint`) ŌĆö the `GOFLAGS` flag is required because the golangci-lint container doesn't have git history.
